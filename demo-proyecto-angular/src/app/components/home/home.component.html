Cree un Componente “punto1” y realice un Slide Personalizado con angular, el cual tendrá Noticias personalizadas a su elección, cada noticia puede estar acompañado por una fotografía descriptiva de la noticia. Como fuente de datos, defina en el Controlador, un array de objetos que será utilizado en la vista HTML. Para recorrer las noticias se dispondrá un botón para adelantar y otro para retrasar.

Cree un componente “punto2” e implemente el Juego del ahorcado. ( utilice bootstrap para maquetar la salida). Consideraciones:
  1. Se dispone de un banco de 10 palabras (array), en una corrida del programa se elige una.palabra. Para simplificar el proceso las palabras son de una categoría especial a elección (Ej. Animales, Jugadores Argentino, Actores Argentinos, etc) que será indicada en alguna parte la pantalla del Juego al usuario.
  2. Por cada corrida:
    1. Los intentos permitidos en cada corrida son a elección (ej. 5). En caso de que en un intento el usuario no adivine a los intentos permitidos se le va restando 1 (uno). El usuario pierde cuando sus intentos llegan a 0(cero). Debe mostrar en un modal que el usuario ha perdido.
    2. En caso de acertar no se le restaran intentos a los disponibles por el usuario y la letra acertada debería ser guardada para poder comparar luego la palabra que se está formando con la que se debe adivinar
  3. Para cada intento se debe elegir una imagen predefinida (a elección) que mostrara el estado del juego (personita) o en la corrida actual.

Una empresa de turismo local quiere crear una app para el proceso de venta de pasajes, se ingresa la información mediante un FORMULARIO almacenándola en un objeto pasaje {{ '{' }}dniPasajero, precioPasaje, categoriaPasajero, fechaCompra{{ '}' }} donde la categoría del pasajero son: m = Menor, a = Adulto, j= Jubilado. Comportamiento:
  1. Si el pasajero es un menor, se deberá hacer un 25% de descuento, si es un jubilado un 50% de descuento, si es adulto no hay descuento. Utilizar Lista Desplegable para elegir la opción del Pasajero, programar el evento (cambio) para hacer el cálculo cuando se cambie en la lista desplegable el tipo de pasajero y también cuando se cambie el precio del pasaje
  2. Utilizar la directiva ngIf para mostrar en una caja LABEL, dentro del mismo formulario, el precio total de la transacción SOLO en el caso de que el precio y el tipo de pasajero hayan sido ingresados. (no utilizar modal).
  3. Mediante un botón se REGISTRAR se registra la venta de los pasajes en una lista (Array). El array será gestionado mediante [ SERVICE DE ANGULAR] para todas las operaciones del CRUD
  4. En una sección inferior al formulario, se debe mostrar mediante una lista (angular-datatable) todos los pasajes vendidos. Además, también en una sección de resumen se debe mostrar un resumen de las ventas de pasajes por categoria de pasajero y el total general.
  5. Utilizar pipes para mostrar información formateada como ser fecha, precio: ej. dd/MM/yyyy
